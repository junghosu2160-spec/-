# 저장소 안내

## 자동 커밋/푸시 정책

이 저장소에서 파일을 수정한 후에는 (사용자가 "수정해줘"라고만 말해도) 별도 확인 없이 다음을 자동으로 수행한다:

1. 변경된 파일을 `git add`
2. 변경 내용을 보고 적절한 커밋 메시지를 직접 작성하여 `git commit`
3. `origin main`으로 `git push`

- 커밋은 각 응답(작업 단위)이 끝날 때 한 번 생성한다 (파일 저장마다 하지 않음).
- 커밋 메시지는 고정 문구가 아니라 실제 diff 내용을 반영해 작성한다.
- 이 저장소는 GitHub 원격(`origin` → `junghosu2160-spec/-`)에 연결되어 있으며 `main` 브랜치를 바로 사용한다.
- 이 승인은 이 저장소 안에서의 일반적인 add/commit/push에 한하며, force push, 브랜치 삭제 등 파괴적 작업에는 적용되지 않는다 (그런 작업은 여전히 별도 확인 필요).
